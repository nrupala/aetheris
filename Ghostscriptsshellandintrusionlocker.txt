The Aetheris Ghost Shell completes the security lifecycle by providing a high-interaction decoy environment. Instead of a simple "Access Denied," suspicious peers are seamlessly routed into an isolated workspace that mimics a real system while logging every action for forensic analysis. [1] 
## 1. Ghost Shell Scripts (aetheris_ghost/scripts)
This script emulates a restricted shell. It provides "fake wins" (e.g., listing attractive but empty files) to keep an intruder engaged. [2, 3] 

#!/bin/sh# simulated_ui.sh - A high-interaction honeypot shell
echo "Welcome to Aetheris Restricted Shell (v0.9.1-beta)"
echo "Unauthorized access is logged."while true; do
    printf "[user@aetheris-ghost ~]$ "
    read cmd
    case $cmd in
        ls) 
            echo "passwords.txt  network_map.json  backups/  confidential_vault_keys.zfs" 
            ;;
        cat*) 
            echo "Permission denied: Accessing $cmd requires Level 4 clearance." 
            ;;
        help) 
            echo "Available commands: ls, cat, help, exit" 
            ;;
        exit) 
            break 
            ;;
        *) 
            echo "sh: $cmd: not found" 
            ;;
    esacdone

## 2. Dummy Workspace Configuration (aetheris_ghost/dummy_vault)
This folder contains "Canary Tokens"—decoy files designed to lure and detect intruders. [4, 5, 6] 

* passwords.txt: Contains fake credentials to monitor brute-force or credential-harvesting attempts.
* network_map.json: A fabricated architecture that leads the intruder to other "Ghost" nodes rather than real production services. [2, 7, 8] 

## 3. Isolation Configuration (docker-compose.ghost.yaml)
This configuration isolates the Ghost Shell into its own network namespace, preventing any lateral movement to the real ZFS vault. [9, 10] 

services:
  aetheris-ghost:
    image: alpine:latest
    container_name: aetheris_ghost_shell
    volumes:
      - ./aetheris_ghost/dummy_vault:/data:ro
      - ./aetheris_ghost/scripts:/scripts:ro
    command: /scripts/simulated_ui.sh
    networks:
      - aetheris_isolation_net
    security_opt:
      - no-new-privileges:true # Hardens the sandbox against privilege escalation
    restart: always
networks:
  aetheris_isolation_net:
    internal: true # Air-gaps the honeypot from external routing
