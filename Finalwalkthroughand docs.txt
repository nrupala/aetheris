## 1. Key Rotation Procedures
Regularly rotating keys is a critical defense-in-depth measure. A 90-day rotation schedule is recommended for static keys.
## A. ZFS Wrapping Key Rotation
You can change the user-facing "Wrapping Key" (passphrase or keyfile) without re-encrypting the data.

   1. Ensure the current key is loaded: zfs load-key aetheris_vault/secure_data.
   2. Rotate the key:
   
   # Rotate to a new hex keyfile
   openssl rand -hex 32 > /root/.aetheris_new_key
   zfs change-key -o keylocation=file:///root/.aetheris_new_key aetheris_vault/secure_data
   
   3. Update your Rust Core environment variables or secrets manager to point to the new location. [1, 2] 

## B. WireGuard Peer Rotation
To rotate a client's access without a complete mesh outage:

   1. Generate New Pair: Run wg genkey for the client.
   2. Update Peers First: Update the Aetheris Hub config with the new Public Key.
   3. Update Client: Swap the Private Key on the mobile/desktop device.
   4. PSK Rotation: Always rotate the Pre-Shared Key (PSK) simultaneously for post-quantum resistance. [3, 4, 5]

   1. Restore ZFS Stream: cat backup.zfs | zfs receive -F aetheris_vault/secure_data.

   3. Unlock: zfs load-key aetheris_vault/secure_data && zfs mount aetheris_vault/secure_data.
