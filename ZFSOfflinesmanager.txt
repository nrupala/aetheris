To ensure your data survives a total site failure, we implement an Offsite & Offline Replicator. This script uses zfs send to transfer encrypted raw blocks. The backup destination receives a data stream it cannot read (since it lacks your master key), but can perfectly store and restore.
1. The Aetheris Offsite Replicator (vault_sync.sh)
This script identifies the most recent local snapshot and "pushes" the delta (the difference since the last backup) to a remote FOSS node.
bash

#!/usr/bin/env bash
# Aetheris Offsite Replicator (Encrypted ZFS Send)
set -e

DATASET="aetheris_vault/secure_data"
REMOTE_USER="backup_agent"
REMOTE_HOST="offsite.node.local" # Or an IP over WireGuard
REMOTE_DATASET="backup_pool/aetheris_mirror"

echo "📡 Initiating Offsite Replication..."

# 1. Identify the latest local snapshot
LATEST_SNAP=$(zfs list -t snapshot -o name -s creation | grep "$DATASET" | tail -n 1)

# 2. Identify the last snapshot successfully sent (check remote)
LAST_SENT=$(ssh $REMOTE_USER@$REMOTE_HOST "zfs list -t snapshot -o name -s creation | grep $REMOTE_DATASET | tail -n 1" | awk -F'@' '{print $2}')

if [ -z "$LAST_SENT" ]; then
    echo "📦 Initializing full recursive encrypted transfer (this may take time)..."
    zfs send -Rwp "$LATEST_SNAP" | ssh $REMOTE_USER@$REMOTE_HOST "zfs receive -F $REMOTE_DATASET"
else
    echo "🏎️ Sending incremental delta: $LAST_SENT -> $LATEST_SNAP"
    zfs send -Rwp -i "$DATASET@$LAST_SENT" "$LATEST_SNAP" | ssh $REMOTE_USER@$REMOTE_HOST "zfs receive -F $REMOTE_DATASET"
fi

echo "✅ Offsite Replication Complete: $LATEST_SNAP"

Use code with caution.
2. The "Offline" Replicator (USB Cold Storage)
For a "Stronger than Cloudflare" air-gapped backup, this script replicates to a physical Cold Storage drive. This is your "Insurance Policy" against ransomware or accidental deletion that might sync across the network.
bash

#!/usr/bin/env bash
# Aetheris Offline/USB Replicator

USB_MOUNT="/mnt/aetheris_cold_storage"
POOL="aetheris_vault"

if mountpoint -q "$USB_MOUNT"; then
    echo "💾 USB Cold Storage detected. Committing snapshots..."
    
    # Send the raw encrypted stream to a file on the USB drive
    TIMESTAMP=$(date +%Y%m%d_%H%M%S)
    zfs send -Rwp "$POOL/secure_data@latest" > "$USB_MOUNT/aetheris_backup_$TIMESTAMP.zfs"
    
    echo "🔒 Offline Backup saved: aetheris_backup_$TIMESTAMP.zfs"
    sync && umount "$USB_MOUNT"
    echo "🔌 Safe to disconnect drive."
else
    echo "❌ Offline drive not mounted at $USB_MOUNT"
fi

Use code with caution.
Why this fulfills the "Sovereign" Vision

    -Rwp Flag: The -R is recursive, -w is raw (keeps it encrypted during transit), and -p keeps properties. This is the "Secret Sauce"—the remote server never sees your plaintext data.
    No Cloud Dependencies: This uses standard SSH and ZFS, FOSS protocols that have existed for decades and will continue to work without a subscription.
    Atomic Integrity: If the transfer is interrupted, zfs receive discards the partial data, ensuring your backup is never "half-baked."
