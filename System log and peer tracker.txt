To finalize the "Sovereign AI Mesh," we implement a Log Aggregator and ID Tracker. This module is the "immune system" of the Aetheris core. It monitors failed OPA decisions and unusual WireGuard handshakes in real-time, automatically blacklisting suspicious Peer IDs. 
Because we are using a Distroless container (which lacks a shell or fail2ban), we must implement this tracking logic directly in the Rust Core using internal state and event-driven logging. 
1. The Zero-Trust Immune System (watcher.rs)
This Rust module tracks "Security Violations" (failed OPA calls) and maintains a volatile blacklist.
rust

use std::collections::HashMap;
use std::sync::Mutex;
use std::time::{Instant, Duration};

// Thresholds for auto-banning
const MAX_FAILURES: u32 = 5;
const BAN_DURATION: Duration = Duration::from_secs(3600); // 1 hour

pub struct SecurityWatcher {
    // Tracks Peer ID -> (Failure Count, Last Seen)
    violations: Mutex<HashMap<String, (u32, Instant)>>,
}

impl SecurityWatcher {
    pub fn new() -> Self {
        Self { violations: Mutex::new(HashMap::new()) }
    }

    pub fn record_failure(&self, peer_id: String) {
        let mut map = self.violations.lock().unwrap();
        let entry = map.entry(peer_id).or_insert((0, Instant::now()));
        entry.0 += 1;
        entry.1 = Instant::now();

        if entry.0 >= MAX_FAILURES {
            println!("🚨 SECURITY: Peer ID {} auto-banned for {} failures.", entry.1, entry.0);
        }
    }

    pub fn is_banned(&self, peer_id: &str) -> bool {
        let map = self.violations.lock().unwrap();
        if let Some((count, last_seen)) = map.get(peer_id) {
            if *count >= MAX_FAILURES && last_seen.elapsed() < BAN_DURATION {
                return true;
            }
        }
        false
    }
}

3. Peer Monitoring & Alerts
To monitor the WireGuard tunnel itself, we use a simple Kernel Debug check to track when peers connect or disconnect. 
Host Monitoring Script (id_tracker.sh):
bash

#!/usr/bin/env bash
# Monitors WireGuard Handshakes for unknown IDs
set -e

# Listen to kernel dynamic debug for WireGuard
echo "module wireguard +p" > /sys/kernel/debug/dynamic_debug/control

tail -f /var/log/kern.log | grep --line-buffered "wireguard" | while read -r line; do
    PEER=$(echo "$line" | grep -oP 'peer \K[0-9]+')
    ENDPOINT=$(echo "$line" | grep -oP 'from \K[0-9.]+')
    
    # Log to Aetheris Auditing
    echo "[$(date +%T)] 🛡️ Handshake Detected: Peer $PEER from $ENDPOINT" >> vault/audit.log
done

Use code with caution.
Why This Beats Standard "Fail2Ban"

    Kernel-Level Integration: By monitoring dyndbg (Dynamic Debug) of the WireGuard module, you see connection attempts before they even reach the application layer.
    Identity over IP: Most firewalls ban IP addresses. Aetheris bans Cryptographic Peer IDs. Even if the attacker switches IPs (VPN, Tor), their banned identity stays blocked.
    Tamper-Proof Audit: Logs are written to the Encrypted ZFS Vault, making them resistant to modification by an intruder trying to hide their tracks.
