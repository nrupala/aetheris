To keep this system ultra-fast and secure, the Rust core will serve SSR (Server-Side Rendered) HTML. We use Standard CSS for the layout, avoiding all JavaScript. This prevents XSS (Cross-Site Scripting) attacks and ensures the UI loads instantly even on low-bandwidth WireGuard connections.
1. The Rust Handler (in main.rs)
Add a simple function to your Rust core to inject data into a raw HTML string. No external templating engine is required for a system this lean.
rust

fn render_dashboard(files: Vec<String>) -> String {
    let mut file_list_html = String::new();
    for file in files {
        file_list_html.push_str(&format!(
            "<li><span>{}</span> <a href='/download/{}' class='btn'>Download</a></li>", 
            file, file
        ));
    }

    format!(
        include_str!("ui/index.html"),
        file_list = file_list_html
    )
}

Use code with caution.

Why this wins in the AI Age

    Semantic Search Form: The <form> tag sends the query directly to your Rust backend. Rust then queries the ChromaDB vector store we set up in the Compose file. No complex API calls or JS fetch() needed.
    Zero-Trust Identity: Because the UI is served through the WireGuard tunnel, the Rust core already knows exactly who you are based on your Peer ID. There is no "Login Screen" because the network layer is the login.
    Speed: There are zero script tags. The page renders the moment the first byte arrives.
    Bot Proof: Since there is no JavaScript to execute, traditional scrapers and automated bots can't interact with the dynamic AI features easily.
